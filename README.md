An interactive AI agent that helps a firm investing in **US airport modernization /
terminal-expansion** projects find the airports where renovation will be most profitable —
judged by rising flight demand pressing against available capacity.

Every figure is **computed deterministically in Python** (never generated by the LLM) and
explained with its assumptions, confidence, and scope. The LLM handles only intent parsing,
airport resolution, tool selection, and translating structured results into natural language.

It answers questions like:

- Which airports in New England are strong candidates for terminal expansion?
- Compare LA and Santa Ana airport congestion levels.
- What percentage of flights out of Anchorage are long-haul?
- What is the unmet flight demand at SFO, and why?

Conversational follow-ups (e.g. _"What about Boston instead?"_) are supported.

## Setup

Requires **Python 3.11+** (developed on 3.14).

1. **Create the environment file** — copy the example and fill in your credentials:

   ```bash
   cp .env.example .env
   ```

   Then edit `.env` and set:
   - `ANTHROPIC_API_KEY` — for the routing/explanation layer.
   - `OPENSKY_CLIENT_ID` / `OPENSKY_CLIENT_SECRET` — OpenSky Network OAuth2 client
     credentials for the dynamic flight-demand engine.

   `.env` is gitignored — real credentials are never committed.

2. **Create a virtual environment and install dependencies:**

   ```bash
   python -m venv .venv
   # Windows (PowerShell):
   .venv\Scripts\Activate.ps1
   # macOS / Linux:
   source .venv/bin/activate

   pip install -r requirements.txt
   ```

## Run

```bash
streamlit run app.py
```

This launches the chat UI in your browser (default `http://localhost:8501`).

## Tests

The deterministic scoring engine is covered by a `pytest` suite (`tests/test_scoring.py`).
The tests are hermetic — both data sources (OurAirports CSVs and the OpenSky API) are
monkeypatched with small fixtures, so they run offline and fast, with no API credits spent.

```bash
pip install -r requirements-dev.txt
pytest
```

They cover the KPI math (traffic, utilization, peak saturation, growth, hourly clipping,
unmet-demand, confidence), the capacity formula's runway-length factors, long-haul
classification (distance + duration fallback), airport name resolution, and the composite
`ExpansionScore` ranking end-to-end (offline).

(`app.py`), an Anthropic tool-calling routing layer (`agent.py`), and a pure-Python
deterministic layer (`test_tool.py`) that owns all data ingestion and math.
