# תוצרים — סוכן בינת השקעות בשדות תעופה

מסמך זה ממפה את התוצרים הנדרשים במטלה למיקום של כל אחד מהם, ועונה ישירות על שלוש שאלות העיצוב.

> **המטלה ביקשה:** קוד מקור, בתוספת מסמך עיצוב/ארכיטקטורה קצר שמסביר (1) מתודולוגיית ניקוד, (2) פשרות מרכזיות, ו-(3) היכן וכיצד נעשה שימוש ב-AI.

## 1. קוד מקור

קוד המקור המלא נמצא במאגר זה:

| קובץ | שכבה | אחריות |
|---|---|---|
| `app.py` | תצוגה | ממשק צ'אט ב-Streamlit + קלט קולי אופציונלי. ללא לוגיקה, ללא חישוב, ללא קריאות AI. |
| `agent.py` | ניתוב | קריאה לכלים (tool-calling) של Anthropic: מבין כוונה, בוחר כלי, מסביר את תוצאת ה-JSON. ללא חישוב. |
| `test_tool.py` | דטרמיניסטית | קליטת נתונים בפייתון טהור + כל חישובי המדדים. מחזיר JSON תקני. ללא AI. |
| `requirements.txt` | — | תלויות מקובעות בגרסה. |
| `.env.example` | — | תבנית לאישורי גישה נדרשים (מפתחות אמיתיים לעולם לא נשמרים ב-Git). |

**הרצה:** העתק `.env.example` ל-`.env` (הוסף מפתחות), הרץ `pip install -r requirements.txt`, ואז `streamlit run app.py`. הוראות מלאות ב-[`../README.md`](../README.md).

### 2א. מתודולוגיית ניקוד

כל מספר מחושב באופן דטרמיניסטי ב-`test_tool.py` ומוחזר כ-JSON — ה-LLM לעולם לא מייצר נתון. הנוסחאות המלאות ב-[`../docs/scoring-and-kpis.md`](../docs/scoring-and-kpis.md).

**המדדים (KPIs)** (משילוב שני מקורות נתונים — תנועות טיסה + קיבולת מסלולים):
- **TrafficVolume** — ממוצע תנועות יומי לאורך חלון של 7 ימים.
- **CapacityIndex** — פרוקסי קיבולת מבוסס מסלולים: `מסלולים שמישים × תפוקה לשעה × שעות פעילות × מקדם אורך`.
- **Utilization** = TrafficVolume / CapacityIndex — כמה השדה מלא.
- **PeakSaturation** = עומס השעה העמוסה / קיבולת שעתית — העומס מורגש בשעות השיא.
- **Growth** — תנועה לעומת בסיס מלפני כ-3 חודשים (מגמה עדכנית, לא שנה־מול־שנה).
- **LongHaulShare** — שיעור ההמראות במרחק ≥ 4,000 ק"מ (מעגל גדול), עם נפילה למשך הטיסה.
- **HourlyClipping** — שיעור שעות הפעילות שמגיעות ל-≥ 90% מהקיבולת השעתית.
- **Confidence** — שיעור הטיסות עם נתונים מלאים; קובע עד כמה השפה ודאית.

**ציון הדירוג (Q1).** עבור קבוצת שדות, כל מדד **מנורמל min-max בתוך הקבוצה** (0–1), ואז משולב בסכום משוקלל שקוף:

```
ExpansionScore = 100 × (0.40·Utilization + 0.30·Growth + 0.20·PeakSaturation + 0.10·LongHaulShare)
```

המשקלים הם הנחות מוצהרות וניתנות לכוונון, המשקפות את תזת ההשקעה: **שדות שכבר מלאים וגם עדיין צומחים הם ההימור הטוב ביותר להרחבה.** אם מדד חסר (למשל אין בסיס צמיחה), הרכיב שלו מושמט, המשקלים הנותרים מנורמלים מחדש כך שיסתכמו ל-1, והביטחון מורד — נתון חסר לעולם לא נחשב כאפס.

**שאלות על שדה בודד (Q2–Q4)** אין להן קבוצה לנרמל מולה, ולכן הן מדווחות **ערכים מוחלטים בתוספת רצועות קטגוריה** (למשל Utilization < 0.50 = נמוך, 0.50–0.85 = בינוני, …) במקום ציון 0–100.

### 2ב. פשרות מרכזיות

בחירות כנות שנעשו כדי להתאים לבנייה של יום אחד, כשכל אחת נחשפת למשתמש במקום להסתתר:

| פשרה | החלטה | למה |
|---|---|---|
| **אין נתוני נוסעים ציבוריים** | פרוקסי לביקוש באמצעות **תנועות טיסה** | תנועות הן האות הציבורי הזמין הטוב ביותר; מסומן כפרוקסי בכל תשובה. |
| **אין נתוני קיבולת שערים/טרמינלים** | פרוקסי לקיבולת מ**מסלולים** (מספר + אורך) | תשתית המסלולים היא אות הקיבולת החינמי והאמין היחיד; פקקי טרמינל אמיתיים עשויים להיות שונים, ואנחנו אומרים זאת. |
| **נתוני OpenSky הם מדגם מבוסס קהל** | לקבל את הדגימה; לכמת אותה עם ציון **Confidence** | מקור ברמת מפקד היה עולה כסף/זמן; מדד הביטחון הופך את אי-הוודאות למפורשת. |
| **מגבלות קרדיט וקצב של OpenSky** | **מטמון בדיסק**, משיכה מקבילה, **מפסק זרם** בעת חניקה, הגבלת גודל אזור | שומר על האפליקציה מהירה ואינטראקטיבית בלי לשרוף את תקציב הקרדיט היומי. |
| **עומק הצמיחה** | בסיס של **כ-90 יום**, לא שנה מלאה | מתאים לתקציב הקרדיט/זמן ולעומק הנתונים שאומת; המגבלה (אין תיקון עונתיות) מצוינת. |
| **פוליש מול מהות** | השקענו במנוע הניקוד + ניתוב ה-AI, לא בעיצוב הממשק | המטלה מעריכה במפורש היגיון על פני פוליש. |

### 2ג. היכן / כיצד נעשה שימוש ב-AI

ה-AI הוא **נתב ומסביר, לעולם לא מחשבון.** זוהי החלטת העיצוב המרכזית.

- **משמש ל:** הבנת כוונת המשתמש, זיהוי שמות/כינויים של שדות, בחירת הכלי הנכון באמצעות **tool-calling** של Anthropic, ותרגום ה-JSON הדטרמיניסטי להסבר ברור בסגנון אנליסט עם הנחות ורמת ביטחון.
- **לעולם לא משמש ל:** חישוב מספר כלשהו. "כלל ברזל" ב-System Prompt אוסר על המודל לצטט, להעריך או לשלוף נתון מהזיכרון — כל ערך כמותי מגיע מפונקציה ב-`test_tool.py`.

**הזרימה:** שאלת משתמש ← `agent.py` (Claude בוחר כלי + פרמטרים) ← `test_tool.py` (חישוב דטרמיניסטי ← JSON) ← `agent.py` (Claude מסביר את ה-JSON) ← תשובה.

זה מבטיח שכל נתון ניתן למעקב עד הקוד, ומשמעו שאפשר להחליף או לשדרג את מודל ה-AI בלי לגעת בלוגיקה העסקית. ראה [`../docs/architecture.md`](../docs/architecture.md) להפרדת השכבות הקפדנית.
